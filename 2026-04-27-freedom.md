Today, South Africans celebrate [Freedom Day](https://en.wikipedia.org/wiki/Freedom_Day_%28South_Africa%29), honoring the first post-apartheid elections in that country.  Over time, it has attracted the counter-programming of [UnFreedom Day](https://en.wikipedia.org/wiki/UnFreedom_Day) to draw attention to the lack of freedom seen by the country's poor.

After rolling out the work last week on adding user interface mock-ups to the blog, which you'll probably see in action this Sunday, it became apparent that the resulting vector images display at strangely small sizes.  The styling forces them to take up two-thirds of the post width, then, and as a vector image, it'll scale cleanly.  It'll still show tiny text for a sufficiently complicated design, but that seems either unlikely for examples in posts.

Somewhat similarly, it finally occurred to me that the language icons behind code segments can obscure the actual code.  If that has bothered you, now mousing over the code will cause the icon to mostly fade out.

Finally, looking back on some of the earliest posts, the [calendar view](/blog/calendar) now removes the "Bicker Progress" prefix that the earliest developer diary posts carried.  While I think that the title prefixes help most cases such as the blog's index and RSS feeds, the calendar view only has about a hundred pixels to identify the posts, so the less recurring information there, the better.  (The Friday posts count less, since anybody using the calendar presumably already knows that it covers social media posts for the previous week, making "Tweets" or "Toots" the only relevant word in the titles.)
