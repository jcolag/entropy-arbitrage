## Entropy Arbitrage, the Code

{% codeberg jcolag/entropy-arbitrage-code %}

From time to time, something peculiar will go wrong when I publish a blog post.  Usually, I can trace the fault to something that I did, such as a typo in a plugin name or when referencing an old post; fixing those and restarting lets me go on with my life.  However, *other* times, the problem appears to come out of nowhere, usually claiming some problem on a post that I haven't touched or even looked at in years, where I "fix" it by restarting the build.

Well, except that, to prevent the publication script from compounding an error by pushing half-built garbage up to my website, the script has some "turn any shady behavior into an error" boilerplate.

```bash
set -o errexit
set -o errtrace
set -o nounset
set -o pipefail
trap 'echo "Aborting due to errexit on line $LINENO. Exit code: $?" >&2' ERR
_ME="$(basename "${0}")"
```

Others probably exist, but this, plus a check on command-line arguments, generally catches any routine problems.  However, this goes a *bit* too far for those odd transient errors, because `git` returns an error when you ask it to commit or push code when it doesn't have any changes to work with.  Re-running the script without changing a post explodes, because...don't try again without changes, apparently.

Avoiding that requires checking `git`'s opinion first, and only committing changes if we can find any changes.

```bash
if ! git diff --quiet || ! git diff --cached --quiet
then
  git commit -m "Automated updates: $(date '+%Y-%m-%d')"
  git push
fi
```
