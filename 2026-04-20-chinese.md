## Mini-Server, Part 24

I never had the time to get that other QR code creator running, so when I found [Mini QR](https://mini-qr-code-generator.vercel.app/), I decided to give it a shot.  Following the [build instructions](https://github.com/lyqht/mini-qr/blob/main/SELF_HOSTING.md#without-docker-) and copying it to one of the servers, it worked fine and immediately.

The QR codes don't look as nice as the other project, but it has niceties such as the data templates that provide an input form and turn the results into a standard text form that fits in a QR code.  For two examples of likely common interest, it can format Wi-Fi network information and vCard information properly, so if I wanted a QR code for people to add me as a contact, the form (minimally) gives me this output.

```
BEGIN:VCARD
VERSION:2.1
N:Colagioia;John;;;
FN:John Colagioia
URL:https://john.colagioia.net
END:VCARD
```

I could always look up the vCard format and write this manually, then double-check all the formatting[^ZSuU4h], sure.  But having it done for me works well.  And I actually had the Wi-Fi connection information as one of my main use cases for something like this---in an emergency where the network goes out, maybe for the neighborhood (not to mention ordinary guests), I'd probably want the ability to get devices on my local network quickly, instead of searching for the name and password and correcting typos---which makes this appealing if not as aesthetically pleasing.

[^ZSuU4h]:  I vaguely remember a blog post where somebody trimmed the fields down to the absolute minimum for conference badges, and editing the text output would allow for reducing the size of the QR code.

## Entropy Arbitrage, the Code

{% codeberg jcolag/entropy-arbitrage-code %}

You'll see two changes to rarely used features.

First, the blog now has a plugin to add a user interface mock-up using [PlantUML's Salt](https://the-lum.github.io/plantuml-doc/en/salt.html) language.  If a post includes something like this...

```
{
  E-Mail   | "me@example.home     "
  Password | "••••••••••••••••    "
  [Cancel] | [  Log In   ]
}
```

That drops something like this image into the post.

{% salt Example adapted directly from the documentation %}
{
  E-Mail   | "me@example.home     "
  Password | "••••••••••••••••    "
  [Cancel] | [  Log In   ]
}
{% endsalt %}

It smells like it might fail a bit on accessibility grounds, which I'll try to squeeze out over time, and it doesn't seem to bother to center anything, but for a first pass, it suffices, in that a person can read every control, plus it gets a container with mouse-over `title` text to set the context.

An upcoming post has a bit speculating on improving a user interface, so it seemed worth finding a tool to do something consistently.  Otherwise, I'd need to pull out something like [Penpot](https://penpot.app/), which seems like overkill.  Personally, I'd prefer it if the syntax leaned more in the descriptive direction, something like Tk's "define a container, then list the items in the container," but this will suffice, and certainly better than writing my own.

On the technical side, the plugin looks a lot like my plugin to handle recipes that called the [Cook](https://cooklang.org/) renderer, here instead calling `plantuml -tsvg -pipe` to take the input from the process (the "pipe" part) and return SVG that it then drops into the post in place of the Star layout.
